# Project Briefing

Generate a comprehensive project briefing document for onboarding new team members.

## Arguments

`$ARGUMENTS` - Optional:
- `--save` - Save to `knowledge/wiki/project-brief.md`

## Examples

```bash
/brief           # Display briefing
/brief --save    # Display and save to wiki
```

## Task

### 1. Gather Project Information

**From `.env`:**
- PROJECT_NAME
- PROJECT_DESCRIPTION

**From `knowledge/project-status/`:**
- Current project status
- Recent decisions
- Active milestones

### 2. Gather People Information

**From `knowledge/people/`:**
- Read all person profiles
- Identify key roles (leads, managers)
- Note team structure

### 3. Gather Active Work

**From `knowledge/tasks/`:**
- Active epics/initiatives
- High priority tasks
- Current blockers

### 4. Gather Key Concepts

**From `knowledge/definitions/`:**
- Important project-specific terms
- Acronyms and their meanings

### 5. Generate Briefing
